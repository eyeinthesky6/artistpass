# Changelog

## v0.1.2 - 2026-07-15

### Added

- A GitHub Pages project site for discovery, setup paths and open-source distribution.
- GitHub Discussions and clearer support/issue routing for community questions.

### Fixed

- Removed incorrect fixed Open Graph dimensions from the portrait social-preview image.
- Clarified that Admin updates native share attachments, while crawler-facing Open Graph tags remain a deployment-time setting.

## v0.1.1 - 2026-07-15

Launch-readiness and contributor-safety hardening.

### Added

- Pinned GitHub Actions for repository validation, secret scanning and CodeQL analysis.
- Weekly Dependabot updates for npm and GitHub Actions dependencies.
- A Gitleaks pre-commit hook, API smoke tests and automated public-asset validation.
- A public agent guide and third-party/sample-asset notices.

### Fixed

- Clean local checkouts no longer request Vercel-only configuration or analytics endpoints while being served locally.
- Contribution and pull-request guidance now uses the same validation command as CI.

## v0.1.0 - 2026-07-11

The first public ArtistPass release.

### Included

- Cinematic fictional actor portfolio with showreel, headshots, role-fit dossier, resume and casting-card flows.
- Browser Admin for editing the profile, previewing changes and publishing live.
- Vercel Blob storage for shared config, image and PDF updates.
- One-click Vercel deployment for developers and an assisted setup path for non-technical artists.
- Vercel Web Analytics and a completed-deployment return page.
- Mobile layout, sharing flows, downloads, SEO basics and a read-only public demo.
- Public screenshots, build stories and open-source contribution guidance.

### Current Limits

- The self-serve deployment path still expects a Git provider and Vercel account.
- Video uploads are not included; artists paste YouTube, Google Drive, Vimeo or other hosted links.
- The public profile is fictional sample material and should be replaced before a real artist publishes a copy.
