## Why Not Put EPK In The Repo Name

`EPK` is a useful keyword, especially for music artists, singers, performers and press-kit searches. It is less plain-language for Indian actors than words like portfolio, profile, showreel, headshots and casting.

So the repo and product stay simple as **ArtistPass**, while EPK stays in keywords, docs, template listing copy and SEO.

## Source Signals Checked

- CastYou uses acting portfolio language around headshots, demo reels, casting directors and digital acting profiles.
- Indian actor portfolio website providers use bio, headshots, resume, video portfolio/showreel, gallery and contact language.
- Marketplace listings for performer websites use portfolio website, actor portfolio website, video/showreel embed and personal website language.
- Music and singer website providers still commonly use EPK, electronic press kit, press kit and media kit.
- Highly starred GitHub repos tend to lead with a plain promise, visible action links, badges, screenshots or banners, quick navigation, contribution guidance and license clarity. ArtistPass should copy that structure, not their volume.

## GitHub README Pattern To Keep

- Banner or logo first.
- One-line promise in plain language.
- Live demo, deploy and docs links above the fold.
- Badges for template, Vercel, no heavy CMS and MIT.
- Quick decision table: who it is for and why they should care.
- Quick start before deep implementation details.
- Screenshots before architecture.
- Contribution and license sections near the end.
