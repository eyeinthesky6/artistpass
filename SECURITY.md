# Security Policy

## Supported Version

Security fixes are made on the latest code in the `main` branch. Older forks and deployments should update from the current template before asking for a fix.

## Report A Vulnerability Privately

Please do not open a public issue for a possible security vulnerability.

Email `6edroid@gmail.com` with the subject `ArtistPass security report` and include:

- what you found;
- the affected page, file or deployment;
- steps that reproduce it;
- the possible impact;
- any suggested fix.

Do not include real passwords, access tokens or private artist material. Replace sensitive values with safe examples.

You should receive an acknowledgement within seven days. Please allow time to confirm and fix the issue before publishing details.

## Deployment Responsibility

Each ArtistPass deployment is owned by the person who creates it. Keep Vercel and GitHub credentials private, use a strong `ADMIN_PUBLISH_PASSWORD`, and replace the fictional sample content before publishing a real artist's site.
